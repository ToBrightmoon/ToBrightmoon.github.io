前言

在上一篇文章中，我们利用源码论证了，状态是 map 下的 sample->pose（base） 。

这篇文章就来说明：在一帧激光数据中，怎么完成预测与更新的。

大致结论

粒子滤波使用蒙特卡罗方法实现贝叶斯滤波框架：用很多带权粒子近似后验。

在 AMCL中，每个粒子都会经历两个过程：

  • 预测

将 odom 系的 delta 输入 motion_model_->odometryUpdate,更新 sample->pose（map）

  • 更新

利用 激光 LaserData + 地图，调用Laser::sensorUpdate → pf_update_sensor，完成sample->weight的更新

与卡尔曼滤波的差别：粒子滤波在更新步 不改状态，只改 权重；最终位姿靠粒子群的统计量给出。
预测里加噪声是粒子滤波的灵魂：不加噪声，所有粒子运动一致，就失去「用样本估计分布」的意义。

每帧激光回调里，先查 base 在 odom 下的位姿 pose：

与上次观测更新后记录的 pf_odom_pose_ 做差，得到 odom 系增量 delta：

  • d_thresh_ / a_thresh_ 来自参数 update_min_d / update_min_a
  • 小车停着：delta 很小 → shouldUpdateFilter 为 false → 不预测、不观测，避免无意义计算

只有 lasers_update_[laser_index]==true 时才调用 odometryUpdate。

差速模型：对每个粒子加噪并改 pose

要点：

  1. 分解：用 odom 的 delta 和 old_pose 算 Rot1–Trans–Rot2。
  2. 加噪：pf_ran_gaussian 让每个粒子的 *_hat 不同 → 粒子云在 map 下散开。
  3. 积分在 map 航向上做：cos(sample->pose.v[2] + ...) 用的是 该粒子自己的 map 朝向，不是 odom 航向（经典 AMCL 近似）。
  4. 循环内没有 sample->weight → 预测 只动 pose。

更新：激光数据怎么变成每个粒子的新权重

updateFilter：ROS 消息 → LaserData

注意：pf_odom_pose_ 在 观测末尾 才更新，所以 delta 是「距上次完整观测」的 odom 位移，不是距上次预测。

sensorUpdate → pf_update_sensor：统一入口

各观测模型（Beam / LikelihoodField / …）都走：

pf_update_sensor 干两件事：调用 sensorFunction 改 weight + 归一化与 w_slow/w_fast。

sensorFunction：以 Beam 模型理解（《概率机器人》光束模型）

对单个粒子 j：

  1. 假设该粒子的 pose 正确 → 在 map 上投影每条 beam；
  2. 比较 实测距离 与 地图射线距离 map_range；
  3. 混合模型得 pz，多束合成 p；
  4. **weight *= p，不碰 pose**。

Nav2 默认似然场（简写）：beam 终点查 occ_dist，高斯似然 + 随机项，同样 p += pz³、weight *= p：

pf_update_sensor：归一化 + 粒子群「健康度」

  • 归一化：所有粒子 weight 之和 = 1
  • **w_slow / w_fast**：长期/短期平均匹配质量；w_fast 突然低于 w_slow 时，重采样会随机撒点

小结

  • 预测：odom delta + 运动模型噪声 → 每个粒子 map 下 pose 不同。
  • 更新：每个粒子用自己的 pose 解释激光 → weight 反映「这个 pose 有多可信」。
  • 合起来才是贝叶斯滤波里的一步「运动更新 + 测量更新」；重采样与位姿输出在后续篇展开。
