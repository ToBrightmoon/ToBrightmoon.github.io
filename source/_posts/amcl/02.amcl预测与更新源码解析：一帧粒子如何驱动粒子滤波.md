## 前言

在第一篇文章中，我们利用源码论证了，状态是 **map 下的 `sample->pose`（base）** 。

这篇文章就来说明：**在一帧激光数据中，怎么完成预测与更新的**。

## 大致结论

![amcl更新流程](/images/amcl/02/amcl_filter_flow.png)

粒子滤波使用蒙特卡罗方法实现贝叶斯滤波框架：用很多带权粒子近似后验。

在 AMCL中，每个粒子都会经历两个过程：
- 预测

> 将 odom 系的 `delta` 输入 `motion_model_->odometryUpdate`,更新 `sample->pose`（map）

- 更新

> 利用 激光 `LaserData` + 地图，调用`Laser::sensorUpdate` → `pf_update_sensor`，完成`sample->weight`的更新

**与卡尔曼滤波的差别**：粒子滤波在更新步 **不改状态**，只改 **权重**；最终位姿靠粒子群的统计量给出。  
**预测里加噪声**是粒子滤波的灵魂：不加噪声，所有粒子运动一致，就失去「用样本估计分布」的意义。

下面按 `laserReceived` 的真实顺序，从源码跟一遍。

## 预测：信息从哪来、怎么作用到每个粒子

### 运动模型插件：`initOdometry`

`on_configure` 里加载车体运动模型（由 `robot_model_type` 决定，默认差速）：

```C++
  motion_model_ = plugin_loader_.createSharedInstance(robot_model_type_);
  motion_model_->initialize(alpha1_, alpha2_, alpha3_, alpha4_, alpha5_);

  latest_odom_pose_ = geometry_msgs::msg::PoseStamped();
```

`alpha1_`～`alpha5_` 控制里程计噪声强度（对应《概率机器人》Table 5.5）。

### `getOdomPose` + `delta`：预测的信息来源

每帧激光回调里，先查 **base 在 odom 下**的位姿 `pose`：

```C++
  if (!getOdomPose(..., laser_scan->header.stamp, base_frame_id_))
```

与**上次观测更新后**记录的 `pf_odom_pose_` 做差，得到 odom 系增量 `delta`：

```C++
bool AmclNode::shouldUpdateFilter(const pf_vector_t pose, pf_vector_t & delta)
{
  delta.v[0] = pose.v[0] - pf_odom_pose_.v[0];
  delta.v[1] = pose.v[1] - pf_odom_pose_.v[1];
  delta.v[2] = angleutils::angle_diff(pose.v[2], pf_odom_pose_.v[2]);

  bool update = fabs(delta.v[0]) > d_thresh_ ||
    fabs(delta.v[1]) > d_thresh_ ||
    fabs(delta.v[2]) > a_thresh_;
  update = update || force_update_;
  return update;
}
```

- `d_thresh_` / `a_thresh_` 来自参数 `update_min_d` / `update_min_a`
- **小车停着**：`delta` 很小 → `shouldUpdateFilter` 为 false → 不预测、不观测，避免无意义计算
- `force_update_`：如 `request_nomotion_update` 服务可强制更新

只有 `lasers_update_[laser_index]==true` 时才调用 `odometryUpdate`。

### 差速模型：对**每个粒子**加噪并改 `pose`

```C++
// 预测步：pose/delta 为 odom 系；sample->pose 为 map 系。
void DifferentialMotionModel::odometryUpdate(
  pf_t * pf, const pf_vector_t & pose,
  const pf_vector_t & delta)
{
  // 取出粒子集合
  pf_sample_set_t * set;

  set = pf->sets + pf->current_set;
  pf_vector_t old_pose = pf_vector_sub(pose, delta);
  
  // 将二维平面上的任意一段运动，分解为三个标准动作：先原地旋转（Rot1），再直线前进（Trans），最后再原地旋转（Rot2）
  double delta_rot1, delta_trans, delta_rot2;
  double delta_rot1_hat, delta_trans_hat, delta_rot2_hat;
  double delta_rot1_noise, delta_rot2_noise;

  if (sqrt(
      delta.v[1] * delta.v[1] +
      delta.v[0] * delta.v[0]) < 0.01)
  {
    delta_rot1 = 0.0;
  } else {
    delta_rot1 = angleutils::angle_diff(
      atan2(delta.v[1], delta.v[0]),
      old_pose.v[2]);
  }
  delta_trans = sqrt(
    delta.v[0] * delta.v[0] +
    delta.v[1] * delta.v[1]);
  delta_rot2 = angleutils::angle_diff(delta.v[2], delta_rot1);

  // 保证向前开和向后开都是噪声都是相同的基数
  delta_rot1_noise = std::min(
    fabs(angleutils::angle_diff(delta_rot1, 0.0)),
    fabs(angleutils::angle_diff(delta_rot1, M_PI)));
  delta_rot2_noise = std::min(
    fabs(angleutils::angle_diff(delta_rot2, 0.0)),
    fabs(angleutils::angle_diff(delta_rot2, M_PI)));

  for (int i = 0; i < set->sample_count; i++) {
    pf_sample_t * sample = set->samples + i;

    // 1. 采样第一阶段旋转（受自身旋转和前进距离影响）
    delta_rot1_hat = angleutils::angle_diff(
      delta_rot1,
      pf_ran_gaussian(
        sqrt(
          alpha1_ * delta_rot1_noise * delta_rot1_noise +
          alpha2_ * delta_trans * delta_trans)));
          
   // 2. 采样平移距离（受前进距离和两阶段旋转量影响）
    delta_trans_hat = delta_trans -
      pf_ran_gaussian(
      sqrt(
        alpha3_ * delta_trans * delta_trans +
        alpha4_ * delta_rot1_noise * delta_rot1_noise +
        alpha4_ * delta_rot2_noise * delta_rot2_noise));
        
   // 3/ 采样第二阶段旋转（受自身旋转和前进距离影响）
    delta_rot2_hat = angleutils::angle_diff(
      delta_rot2,
      pf_ran_gaussian(
        sqrt(
          alpha1_ * delta_rot2_noise * delta_rot2_noise +
          alpha2_ * delta_trans * delta_trans)));

    // 更新粒子状态
    sample->pose.v[0] += delta_trans_hat *
      cos(sample->pose.v[2] + delta_rot1_hat);
    sample->pose.v[1] += delta_trans_hat *
      sin(sample->pose.v[2] + delta_rot1_hat);
    sample->pose.v[2] += delta_rot1_hat + delta_rot2_hat;
  }
}
```

**要点**：

1. **分解**：用 odom 的 `delta` 和 `old_pose` 算 Rot1–Trans–Rot2。
2. **加噪**：`pf_ran_gaussian` 让每个粒子的 `*_hat` 不同 → 粒子云在 map 下散开。
3. **积分在 map 航向上做**：`cos(sample->pose.v[2] + ...)` 用的是 **该粒子自己的 map 朝向**，不是 odom 航向（经典 AMCL 近似）。
4. **循环内没有 `sample->weight`** → 预测 **只动 pose**。

## 更新：激光数据怎么变成每个粒子的新权重

### `updateFilter`：ROS 消息 → `LaserData`

```c++
bool AmclNode::updateFilter(
  const int & laser_index,
  const sensor_msgs::msg::LaserScan::ConstSharedPtr & laser_scan,
  const pf_vector_t & pose)
{
  // 重新计算出在机器人视角下，雷达扫描的真实起始角度 angle_min 和两点间的真实角度增量
  nav2_amcl::LaserData ldata;
  ldata.laser = lasers_[laser_index];
  ldata.range_count = laser_scan->ranges.size();
  
  geometry_msgs::msg::QuaternionStamped min_q, inc_q;
  min_q.header.stamp = laser_scan->header.stamp;
  min_q.header.frame_id = nav2_util::strip_leading_slash(laser_scan->header.frame_id);
  min_q.quaternion = orientationAroundZAxis(laser_scan->angle_min);

  inc_q.header = min_q.header;
  inc_q.quaternion = orientationAroundZAxis(laser_scan->angle_min + laser_scan->angle_increment);
  try {
    tf_buffer_->transform(min_q, min_q, base_frame_id_);
    tf_buffer_->transform(inc_q, inc_q, base_frame_id_);
  } catch (tf2::TransformException & e) {
    RCLCPP_WARN(
      get_logger(), "Unable to transform min/max laser angles into base frame: %s",
      e.what());
    return false;
  }
  double angle_min = tf2::getYaw(min_q.quaternion);
  double angle_increment = tf2::getYaw(inc_q.quaternion) - angle_min;

  // wrapping angle to [-pi .. pi]
  angle_increment = fmod(angle_increment + 5 * M_PI, 2 * M_PI) - M_PI;

  // 去掉太近和太远的点
  if (laser_max_range_ > 0.0) {
    ldata.range_max = std::min(laser_scan->range_max, static_cast<float>(laser_max_range_));
  } else {
    ldata.range_max = laser_scan->range_max;
  }
  double range_min;
  if (laser_min_range_ > 0.0) {
    range_min = std::max(laser_scan->range_min, static_cast<float>(laser_min_range_));
  } else {
    range_min = laser_scan->range_min;
  }

  // 构建 amcl内部数据结构
  ldata.ranges = new double[ldata.range_count][2];
  for (int i = 0; i < ldata.range_count; i++) {
    if (laser_scan->ranges[i] <= range_min) {
      ldata.ranges[i][0] = ldata.range_max;
    } else {
      ldata.ranges[i][0] = laser_scan->ranges[i];
    }
    // Compute bearing
    ldata.ranges[i][1] = angle_min +
      (i * angle_increment);
  }
  lasers_[laser_index]->sensorUpdate(pf_, reinterpret_cast<nav2_amcl::LaserData *>(&ldata));
  lasers_update_[laser_index] = false;
  pf_odom_pose_ = pose;
  return true;
}
```

注意：`pf_odom_pose_` 在 **观测末尾** 才更新，所以 `delta` 是「距上次**完整观测**」的 odom 位移，不是距上次预测。

### `sensorUpdate` → `pf_update_sensor`：统一入口

各观测模型（Beam / LikelihoodField / …）都走：

```c++
pf_update_sensor(pf, (pf_sensor_model_fn_t) sensorFunction, data);
```

`pf_update_sensor` 干两件事：**调用 `sensorFunction` 改 weight** + **归一化与 w_slow/w_fast**。

### `sensorFunction`：以 Beam 模型理解（《概率机器人》光束模型）

```C++
  for (j = 0; j < set->sample_count; j++) {
    sample = set->samples + j;
    pose = sample->pose;
    pose = pf_vector_coord_add(self->laser_pose_, pose);  // map 下雷达位姿

    p = 1.0;
    for (i = 0; i < data->range_count; i += step) {
      map_range = map_calc_range(self->map_, pose.v[0], pose.v[1],
        pose.v[2] + obs_bearing, data->range_max);
      z = obs_range - map_range;
      pz += z_hit * exp(-z²/2σ²);   // 击中
      ...                            // 短读、丢失、随机噪声
      p += pz * pz * pz;             // 立方和聚合多光束（ad-hoc，但好用）
    }
    sample->weight *= p;
    total_weight += sample->weight;
  }
  return total_weight;
```

对**单个粒子** `j`：

1. 假设该粒子的 `pose` 正确 → 在 map 上投影每条 beam；
2. 比较 **实测距离** 与 **地图射线距离** `map_range`；
3. 混合模型得 `pz`，多束合成 `p`；
4. **`weight *= p`**，**不碰 `pose`**。

### `pf_update_sensor`：归一化 + 粒子群「健康度」

```c++
void pf_update_sensor(pf_t * pf, pf_sensor_model_fn_t sensor_fn, void * sensor_data)
{
  total = (*sensor_fn)(sensor_data, set);

  if (total > 0.0) {
    for (i = 0; i < set->sample_count; i++) {
      sample->weight /= total;    // 归一化
      w_avg += sample->weight;
    }
    w_avg /= set->sample_count;
    pf->w_slow += pf->alpha_slow * (w_avg - pf->w_slow);
    pf->w_fast += pf->alpha_fast * (w_avg - pf->w_fast);
  } else {
    sample->weight = 1.0 / set->sample_count;  // 全失效则均匀
  }
}
```

- **归一化**：所有粒子 weight 之和 = 1
- **`w_slow` / `w_fast`**：长期/短期平均匹配质量；`w_fast` 突然低于 `w_slow` 时，重采样会随机撒点


## 小结

![summary](/images/amcl/02/amcl-filter.png)

- **预测**：odom `delta` + 运动模型噪声 → 每个粒子 **map 下 pose** 不同。
- **更新**：每个粒子用自己的 pose 解释激光 → **weight** 反映「这个 pose 有多可信」。
- **合起来**才是贝叶斯滤波里的一步「运动更新 + 测量更新」；重采样与位姿输出在后续篇展开。
